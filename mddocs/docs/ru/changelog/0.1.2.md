# 0.1.2 (2024-04-15) { #0.1.2 }

## Новые функции

- Переименован модуль `syncmaster.backend.main` в `syncmaster.backend`. Теперь он принимает те же аргументы, что и `uvicorn`. ([#35](https://github.com/MobileTeleSystems/syncmaster/issues/35))
- Добавлен модуль `syncmaster.db.migrations` для запуска `alembic` с правильной конфигурацией. ([#35](https://github.com/MobileTeleSystems/syncmaster/issues/35))
- Изменены образы бэкенда и рабочего процесса для использования точки входа. ([#35](https://github.com/MobileTeleSystems/syncmaster/issues/35))

## Улучшения

- Запуск миграций базы данных в точке входа образа бэкенда, перед запуском сервера бэкенда. ([#35](https://github.com/MobileTeleSystems/syncmaster/issues/35))
- Добавлены проверки работоспособности в пример `docker-compose.yml`. ([#35](https://github.com/MobileTeleSystems/syncmaster/issues/35))
