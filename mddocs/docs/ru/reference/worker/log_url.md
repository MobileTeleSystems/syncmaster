# Установка значения `Run.log_url` { #worker-log-url }

Каждый запуск в системе связан с URL-адресом журнала, по которому доступны логи рабочего процесса(воркера) Celery.
Этот URL-адрес журнала может указывать на экземпляр Elastic или другой инструмент логирования, например, Grafana.

URL-адрес журнала генерируется на основе шаблона, настроенного в конфигурации.
Параметр конфигурации:

```bash
SYNCMASTER__WORKER__LOG_URL_TEMPLATE=https://grafana.example.com?correlation_id=\{\{ relationships_id \}\}&run_id=\{\{ run.id \}\}
```

В этом примере журналы запуска можно получить либо по идентификатору корреляции `x-request-id` в http-заголовках, либо по значению поля `Run.Id`.
